## Contributing

Useful contribution paths:

- improve documentation clarity
- contribute examples and recipes
- report SDK issues
- propose runtime-specific improvements

## Communication Shape

The project site should remain the canonical entry point. Repositories hold implementation details, while the website explains how the parts fit together.
