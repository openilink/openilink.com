## 为什么是 OpenILink

OpenILink 不是单一仓库，而是一套围绕会话接入构建的公开表面。它把最常用的开发动作压缩成一条清晰链路：登录、收消息、回消息、维护上下文、再往上叠加更高层能力。目标很直接，让团队在理解接口之前，先看到一条可执行的工程路径。

## 接入原则

OpenILink 倾向于保持窄入口、强一致性和低理解成本：

- 对外接口尽量对齐不同语言 SDK
- 文档贴近真实实现，不写空泛概念页
- 示例以“可以直接改成自己项目”为目标
- 允许生态逐步扩展，而不是一开始就设计过重
